# auto-ideas

Claude Codeで自律的に情報収集・アイデア生成・レビュー・改善を繰り返すシステム。

## 仕組み

```
/loop 起動
    │
    ▼
🔍 人格A「リサーチャー」
    テーマ選定 → WebSearch → アイデア3件生成
    │
    ▼
🧐 人格B「クリティック」
    市場規模・競合・実行可能性・ビジネスモデルを厳しくレビュー
    │
    ▼
✏️ 人格C「エディター」
    レビューを反映して改善案を作成
    │
    ▼
ideas/YYYY-MM-DD_テーマ名.md に保存
    │
    └──→ 次のサイクルへ（自動）
```

## セットアップ（別PCでの再現手順）

### 1. 前提条件
- [Claude Code](https://claude.ai/code) がインストール済みであること
- GitHubアカウント・Linearアカウントがあること

### 2. リポジトリのクローン

```bash
git clone https://github.com/<YOUR_USERNAME>/auto-ideas.git
cd auto-ideas
```

### 3. Claude Codeで開く

```bash
claude
```

### 4. ループを起動

Claude Codeのチャットに以下を貼り付ける：

```
/loop LOOP_PROMPT.mdを読んで、その手順通りに実行してください。作業ディレクトリは <このリポジトリのパス> です。
```

### 5. Linearへの連携（任意）

アイデアファイルが生成されたら、以下のプロンプトでLinearにイシューを作成できる：

```
ideas/ フォルダの最新ファイルを読んで、改善版アイデアをLinearのイシューとして登録してください。
```

## ファイル構成

| ファイル | 役割 |
|---|---|
| `themes.md` | テーマ一覧（自由に追加・編集可能） |
| `LOOP_PROMPT.md` | ループ用プロンプト（3人格の定義） |
| `ideas/` | 生成されたアイデアが蓄積されるフォルダ |

## テーマの追加方法

`themes.md` に箇条書きで追加するだけで、次のサイクルから自動的に対象になる。

## 人格の説明

| 人格 | 役割 | 特徴 |
|---|---|---|
| 🔍 リサーチャー | 情報収集・アイデア生成 | 最新情報をWebSearchで収集し、実行可能なアイデアを3件出す |
| 🧐 クリティック | 厳しいレビュー | 市場・競合・実行可能性・ビジネスモデルを問い詰める。甘い評価禁止 |
| ✏️ エディター | 改善案の作成 | クリティックの指摘を反映して磨き上げる |
